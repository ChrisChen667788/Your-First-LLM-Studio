# v1.3.0 MCP and Secure Extension Acceptance

Date: 2026-07-19
Scope: local implementation and real Apple Silicon acceptance
Local status: **PASS**
Production status: **HOLD**

## English

### What shipped

- A persisted MCP server registry that accepts the allowlisted, lockfile-pinned official filesystem server and verifies its npm integrity plus entrypoint digest.
- Real MCP stdio initialization, capability discovery, tool annotations, and tool execution through the official TypeScript SDK.
- Ed25519-signed extension install, update, disable/enable, and rollback with atomic activation and durable lifecycle receipts.
- Per-extension permission grants, secret scopes, confirmation boundaries, quarantine review, dependency planning, and path-safe bundle expansion.
- OS-enforced macOS Seatbelt execution that strips extension secrets and denies filesystem writes and network access.
- A unified acceptance runner, `/experiments` evidence panel, promotion read model, route smoke, and 2x screenshot evidence.

### Real workload

The acceptance runner launched `@modelcontextprotocol/server-filesystem@2026.7.10` over stdio under macOS Seatbelt. MCP initialization and `tools/list` returned 14 tools: 10 read-only and 3 destructive. Reading the bounded acceptance marker succeeded; `write_file` was denied and left no artifact. The standalone sandbox rehearsal also confirmed denied writes, denied network access, and absent broker-only secrets.

The signed extension lifecycle installed `1.0.0`, updated to `1.1.0`, and rolled back to `1.0.0`. A tampered payload entered quarantine, a signed path-traversal bundle was rejected before activation, and a missing dependency kept the install plan blocked. Permission grant/revoke, secret scope, and quarantine review receipts passed.

Totals: **11/11 local security checks PASS**, **1/1 registered MCP server PASS**, **14 tools discovered**, and **macOS Seatbelt OS enforcement PASS**.

### Honest production boundary

Production promotion remains HOLD until all of these external receipts exist:

1. An independently managed community publisher trust root and signing receipt.
2. A real Linux bubblewrap or container-isolation receipt.
3. A real Windows sandbox acceptance receipt.
4. A remote MCP Streamable HTTP OAuth lifecycle receipt.

The ephemeral internal acceptance key and the local filesystem server prove the product contract; they are not represented as independent community or cross-platform production evidence.
