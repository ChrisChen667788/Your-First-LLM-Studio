# v1.0.0 GA Evidence Bundle

Date: 2026-07-10

## Scope

This slice creates one durable manifest for the non-cloud GA closeout while preserving a separate production-readiness result. Local deployment rehearsal is not treated as cloud KMS, workload identity, or immutable archive sign-off.

## Added Contracts

- `provider.ops-evidence-snapshots.v1` stores Provider Ops snapshots without credentials or raw prompts.
- Only snapshots backed by a successful remote chat or release probe can be pinned.
- Pinned snapshots use the underlying success timestamp and cannot extend the 24-hour freshness window.
- `admin.compatibility-deletion-manifest.v1` evaluates every deprecated wrapper and canonical replacement.
- Route smoke writes the latest report plus timestamped history reports and consecutive-pass metrics.
- `experiments.ga-release-evidence-bundle.v1` groups promotion, route smoke, compatibility, Provider snapshots, public release, and production control-plane evidence.

## Product Surfaces

- `/admin` Provider Ops owns release probe, snapshot, pin, delete, retention, JSON export, and Markdown export controls.
- `/admin` compatibility sunset cards show keep, delete-ready, or already-removed decisions per wrapper.
- `/experiments` shows non-cloud and production readiness side by side and can write the current GA bundle.
