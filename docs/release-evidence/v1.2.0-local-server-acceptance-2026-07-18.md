The first real run honestly retained `13/15`: bounded concurrency had two successful HTTP/usage responses whose output remained in the model's reasoning field, and the immediate `/api/ps` read raced Ollama's asynchronous unload. The implementation was corrected to use the documented OpenAI-compatible `reasoning_effort: "none"` field and bounded residency polling. The final run reduced completion work from 143 to 24 tokens and average recorded latency from 546 ms to 179 ms.

### Promotion boundary

Local acceptance is `PASS`. Production promotion remains `HOLD` for two external proofs:

1. Authenticated LAN traffic from a separately trusted device.
2. A sustained idle-eviction daemon window rather than a deterministic dry-run.

These are not inferred from the local process run. Machine-readable evidence is in [`v1.2.0-local-server-acceptance-2026-07-18.json`](./v1.2.0-local-server-acceptance-2026-07-18.json), and the refreshed `3840x2400` UI evidence is [`models-studio.png`](../assets/screenshots/models-studio.png).

## 中文

本轮不是静态 contract 演练，而是直接调用本机 Ollama `0.31.1` 和真实 `qwen3:0.6b`。最终 15 个切片全部通过，覆盖真实模型发现、注册、预热、内存驻留、OpenAI-compatible models/chat、SSE 流式、双并发、token/latency ledger、API key 归因与撤销、LAN 安全策略、日志脱敏与保留、排空切换/回滚策略、idle eviction dry-run，以及模型真实卸载后重新加载。

第一轮诚实记录为 `13/15`：并发 HTTP 和 usage 实际成功，但 Qwen3 在 32-token 预算内只输出 reasoning；卸载也成功，但 `/api/ps` 需要约 1 秒才收敛。最终实现改用 Ollama 官方支持的 `reasoning_effort: "none"`，并对驻留状态增加有界轮询。相同验收的 completion tokens 从 143 降到 24，平均记录延迟从 546 ms 降到 179 ms。

本地验收为 PASS，生产 promotion 仍保持 HOLD：还需要一台独立可信设备的 authenticated LAN 请求证据，以及真实长时 idle daemon window。仓库不会用本机策略测试替代这两个外部签收。
