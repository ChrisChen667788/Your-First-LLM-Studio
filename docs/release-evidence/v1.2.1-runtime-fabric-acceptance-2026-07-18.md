## English

### What shipped

- One backend-neutral contract for `health`, `discover`, `chat`, `stream`, `prewarm`, `unload`, and `cancel`.
- Six implemented adapters: MLX, Ollama, llama.cpp, LocalAI, vLLM, and SGLang.
- Every one of the 42 backend/operation decisions returns either a normalized success or an actionable normalized rejection.
- Host and model compatibility checks reject unsupported platform, accelerator, format, and memory combinations before download or execution.
- The MLX Gateway now honors `stream: true` on `/v1/chat/completions` with OpenAI-compatible SSE and `[DONE]`; the legacy NDJSON stream route remains available.
- A reproducible evidence runner resolves the GGUF backing the Ollama model, launches a temporary `llama-server`, runs the shared suite, and shuts the process down.

This release does not convert deterministic routing rehearsals or unavailable endpoints into production evidence.

## 中文

### 本轮完成

- 建立统一 `health / discover / chat / stream / prewarm / unload / cancel` runtime 合同。
- MLX、Ollama、llama.cpp、LocalAI、vLLM、SGLang 六个 adapter 全部实现，移除 planned descriptor。
- 42 个 backend/operation 组合全部返回标准化成功或带错误码的标准化拒绝。
- 在下载和执行前检查平台、加速器、模型格式与内存，避免不兼容任务进入运行态。
- MLX Gateway 的标准 `/v1/chat/completions` 已支持 `stream: true`、SSE 与 `[DONE]`，旧 NDJSON 路径继续兼容。
- 证据脚本会从 Ollama 模型解析真实 GGUF，临时启动 llama.cpp，完成验收后自动关闭。

### 真实结果

本机为 Apple M1 Max、Darwin arm64、32 GB 统一内存。MLX、Ollama 和 llama.cpp 三个真实进程全部通过健康、模型发现、非流式 marker、OpenAI-compatible SSE 与 usage 归一化。汇总结果为 3/3 实时后端、6/6 adapter contract、42/42 operation decision，平均请求延迟 404 ms。

当前主机上的 MLX、Ollama、llama.cpp 与 LocalAI profile 满足硬件合同；vLLM 与 SGLang 因缺少 Linux/NVIDIA 条件，在网络请求前以 `platform_unsupported` 和 `accelerator_unavailable` 明确拒绝。

生产 promotion 仍然需要真实 LocalAI、Linux/NVIDIA vLLM/SGLang，以及独立机器的异构 remote-node failover receipt。本地闭环不会被描述成生产签收。
