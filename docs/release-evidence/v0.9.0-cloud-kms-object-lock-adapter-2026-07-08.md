# v0.9.0 Cloud KMS and Object Lock Adapter

Date: 2026-07-08

## Scope

This follow-up replaces the v0.9 local signer/archive rehearsal with a production adapter boundary. Local evidence can still support development, but production sign-off now requires:

- AWS KMS or HSM-backed signing through `aws kms sign`.
- AWS KMS verification through `aws kms verify`.
- Immutable audit archive through S3 Object Lock retention.
- A failover rehearsal that ties together the usage outbox record, S3 Object Lock audit record, and verified AWS KMS receipt.

## Implementation

- `deployment.control-plane.v1` now exposes both `localReadiness` and `productionReadiness`.
- `/api/deployment` still supports local development rehearsal, but `productionReadiness` only passes when cloud evidence exists.
- `POST /api/deployment` accepts `requireCloud: true` to fail closed when cloud configuration is missing or the cloud command fails.
- The release evidence matrix scores `v0.9.0` from `productionReadiness`, not from local rehearsal evidence.
- The control-plane read-model now reports:
  - `controlPlane.cloud.configured`
  - `controlPlane.cloud.provider`
  - `controlPlane.auditArchive.immutableArchivedEvents`
  - `controlPlane.kmsSigning.verifiedCloudReceipts`
  - `productionReadiness.blockers`

## Required Configuration

Preferred secure manifest:

```json
{
  "cloudProvider": "aws",
  "awsCliPath": "~/Library/Python/3.11/bin/aws",
  "awsRegion": "<region>",
  "awsProfile": "<profile>",
  "kmsKeyId": "arn:aws:kms:<region>:<account-id>:key/<key-id>",
  "auditS3Bucket": "<object-lock-enabled-bucket>",
  "auditS3Prefix": "first-llm-studio/deployment-audit",
  "objectLockMode": "COMPLIANCE",
  "retentionDays": 365,
  "kmsSigningAlgorithm": "RSASSA_PSS_SHA_256",
  "requireCloud": true
}
```

Default manifest locations:

- `/secure/runtime/aws-production-promotion.json`
- `/secure/runtime/aws-deployment-control-plane.json`

Override path:

```bash
export FIRST_LLM_DEPLOYMENT_CONFIG="/secure/runtime/aws-production-promotion.json"
```

Environment-only fallback:

```bash
export FIRST_LLM_DEPLOYMENT_CLOUD_PROVIDER=aws
export FIRST_LLM_AWS_CLI_PATH="$HOME/Library/Python/3.11/bin/aws"
export FIRST_LLM_AWS_KMS_KEY_ID="arn:aws:kms:<region>:<account-id>:key/<key-id>"
export FIRST_LLM_AUDIT_S3_BUCKET="<object-lock-enabled-bucket>"
export FIRST_LLM_AUDIT_S3_PREFIX="first-llm-studio/deployment-audit"
export FIRST_LLM_AUDIT_OBJECT_LOCK_MODE="COMPLIANCE"
export FIRST_LLM_AUDIT_RETENTION_DAYS="365"
export FIRST_LLM_AWS_KMS_SIGNING_ALGORITHM="RSASSA_PSS_SHA_256"
```

Optional:

```bash
export FIRST_LLM_AWS_REGION="<region>"
export FIRST_LLM_AWS_PROFILE="<profile>"
export FIRST_LLM_DEPLOYMENT_REQUIRE_CLOUD=1
```

## Preflight

```bash
npm run preflight:aws-production-promotion -- \
  --config /secure/runtime/aws-production-promotion.json
```

Dry-run config parsing without touching AWS:

```bash
npm run preflight:aws-production-promotion -- \
  --config /secure/runtime/aws-production-promotion.json \
  --dry-run --json
```

The preflight checks:

- AWS CLI launch path.
- `sts get-caller-identity`.
- KMS key usage is `SIGN_VERIFY`.
- KMS key supports the configured signing algorithm.
- S3 bucket has Object Lock enabled.
