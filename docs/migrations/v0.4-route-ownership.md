# v0.4 Route Ownership Migration

Date: 2026-06-12

## What Changed

- Daily Fine-tune, Compare, Models, Benchmarks, Retrieval, and Experiments workflows now have foreground routes.
- `/admin` is a governance surface for monitoring, configuration, health, audit, and compatibility handoffs.
- Feature routes call canonical APIs owned by their feature application boundaries.
- Legacy admin APIs remain available as compatibility wrappers during the v0.4 transition.

## Route Map

| Previous entry | Canonical v0.4 entry | Compatibility status |
| --- | --- | --- |
| Admin Fine-tune mirror | `/fine-tune` | Admin handoff only |
| Agent-owned Compare shell | `/compare` | Embedded Compare remains available through feature ports |
| Admin model discovery | `/models` | Admin component/API wrappers remain temporarily |
| Admin benchmark controls | `/benchmarks` | Admin retains governance history/evidence |
| Admin knowledge base | `/retrieval` | `/api/admin/knowledge-base*` remains a thin wrapper |
| Admin timeline | `/experiments` | `/api/admin/timeline` remains a thin wrapper |

## Canonical APIs

- Retrieval document/import operations: `/api/retrieval`
- Retrieval validation: `/api/retrieval/query`
- Experiment timeline and retention: `/api/experiments`
- Model discovery/install verification: `/api/models/discovery`

## Operator Notes

- Existing local data files are reused; this migration does not rewrite user datasets, sessions, adapters, benchmark history, or timeline JSONL records.
- Bookmarks to legacy admin APIs continue to work during the compatibility window.
- New frontend work should import feature contracts/actions rather than admin components or deep persistence modules.
- Retention is explicit and user-triggered. Failure/conflict and artifact-bearing timeline events are protected by default.

## Rollback

The compatibility wrappers allow a source rollback without a data migration. If a foreground route is temporarily unavailable, use the corresponding Admin governance handoff and verify the canonical API response before changing local data.
