# v0.4.1 - 2026-07-01

First LLM Studio v0.4.1 is the stability and evidence baseline after the v0.4 foreground product refactor. It keeps the product IA stable, closes the Fine-tune/LoRA professionalism gaps, and refreshes the public GitHub/ModelScope evidence with high-resolution screenshots.

## Scope

- Stabilize the foreground Agent, Fine-tune, Models, Benchmarks, Retrieval, Experiments, and Admin routes after the v0.4 migration.
- Turn LoRA standards from guidance into a reusable recipe contract and worker execution chain.
- Publish real local LoRA evidence with checkpoint selection, reports, charts, and reproducible manifests.
- Keep GitHub and ModelScope synchronized from the same committed Git tree.

## Included

### Fine-tune and LoRA

- LoRA recipe state now carries target modules, scheduler, warmup ratio, packing policy, eval cadence, save cadence, best-checkpoint metric, and load-best-at-end policy through the UI state, saved API payloads, YAML preview, command preview, and worker bundle.
- Professional defaults now align around `saveEverySteps: 100`, avoiding the previous mismatch between recommendation copy and recipe form defaults.
- The worker materializes checkpoint directories, records checkpoint events, tracks best checkpoint selection, and includes the chosen checkpoint in report/chart/export evidence.
- Real Qwen3 4B LoRA evidence is archived under [`docs/release-evidence/finetune-qwen4b-lora-2026-07-01`](../release-evidence/finetune-qwen4b-lora-2026-07-01).
- The 4B run used `mlx-community/Qwen3-4B-Instruct-2507-4bit`, 960 starter samples, 816 training steps, eval/save every 100 steps, peak memory 3.316 GB, and selected checkpoint step 800 with `eval_loss=0.066`.

### Charts and Release Evidence

- Fine-tune chart evidence now includes train/validation curves, save markers, eval markers, best-checkpoint marker, smoothing metadata, raw metrics export, and the selected checkpoint path.
- Public README assets include a 3360x1960 PNG exported from the chart SVG plus the original vector SVG for sharp zooming.
- Existing route screenshots remain 3200x2000 at 2x DPR for GitHub and ModelScope display.

### Runtime, Benchmark, and Admin

- Agent runtime gateway detection reports the chosen Python runtime instead of assuming `python3.12`.
- Route smoke requests have explicit timeouts, and empty Benchmark progress now returns an idle snapshot instead of an error-shaped payload.
- Benchmark runs skip fatal unavailable remote targets such as missing key, auth, model-not-found, quota, and billing failures rather than stalling the whole run.
- Remote runtime status now reports unconfigured API targets as HTTP 200 with `available=false`, so provider status scans do not turn missing keys into frontend-breaking 500s.
- Admin compatibility wrappers now emit usage evidence so deprecated Admin APIs can be removed after the sunset gate with observed traffic data.
- Model Hub exposes runtime profiles, idle-unload configuration, request logs, and storage paths through the runtime operations API.
- Runtime dependencies are aligned on `next@15.5.18`, `eslint-config-next@15.5.18`, and `postcss@8.5.16` with an npm override so the production audit path no longer points at the old Next 14/PostCSS advisory baseline.
