# v0.4.2 - 2026-07-02

First LLM Studio v0.4.2 is a release-evidence hygiene patch on top of the v0.4.1 stability baseline.

## Scope

- Record the GitHub and ModelScope synchronization evidence after the real Qwen3 4B LoRA release run.
- Keep public screenshots high resolution while avoiding ModelScope clone-time LFS pointer fallbacks for README-facing images.
- Establish the first v0.5.0 development checkpoint for Provider Health Desk v2 and Adapter Export wizard without promoting the whole v0.5 track yet.

## Included

### Screenshot And ModelScope Evidence

- Re-synced the committed GitHub tree to ModelScope using the ModelScope SDK after HTTPS Git credentials were unavailable in the local shell.
- Verified ModelScope `main` and `master` branches both contain the release tree and the Qwen3 4B LoRA evidence assets.

### v0.5.0 First Development Slice

- Provider Health Desk v2 now projects provider status, success/failure rate, trend buckets, model-level cost/latency, profile-level policy view, and actionable provider policy recommendations from existing chat and connection-check logs.
- Adapter Export wizard now records publish target, license review, dataset attribution review, secret scan status, sample prompts, and known limitations in the UI, command preview, export manifest, report, and `PUBLISH_CHECKLIST.md`.
- The next 10 release train is now a typed Experiments contract, available through `/api/experiments/release-train` and visible on `/experiments`.
- Model Hub runtime operations now expose `models.runtime-operations.v2`, capability flags, Developer API guide fields, OpenAI-compatible chat/models URLs, key status, token accounting fields, latency fields, and a Developer API panel in `/models`.
- Provider Health Desk rows now expose retry/timeout policy templates with provider kind, first-token timeout, total timeout, stream idle timeout, retry budget, fallback profile, and a suggested template rendered in Admin.
- Benchmark release evidence now exposes `benchmark.release-evidence-summary.v1`, grouping pinned runs into release-note-ready summaries with missing-run, target coverage, failed/skipped sample, success rate, and source-group details rendered in `/benchmarks` and Admin.
- Model Hub runtime operations now expose target cards with endpoint, key status, runtime profile coverage, request count, token total, latency, and idle-unload policy, rendered in the primary `/models` workspace.
- Provider Ops now exposes `provider.ops-evidence-summary.v1`, grouping Provider Health Desk rows into release-gate source notes with status counts, action/watch counts, failure classes, cost/token totals, retry template summaries, and top risks.
