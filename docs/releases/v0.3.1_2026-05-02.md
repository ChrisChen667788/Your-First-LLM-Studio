# v0.3.1 - 2026-05-02

First LLM Studio v0.3.1 focuses on making the local fine-tune workflow more complete, easier to audit, and safer to share before public release.

## Highlights

- Fine-tune report preview page for completed jobs.
- One-click full fine-tune bundle download with reports, metrics, manifest, logs, config, dataset snapshot, and adapter artifacts.
- Same-adapter multi-run loss overlay for comparing training runs.
- Expanded beginner-friendly long-run dataset presets for local fine-tuning.
- More robust community dataset conversion across chat, conversation, turn-based, and instruction/output formats.
- Compact Compare small-screen runtime rail behavior to reduce layout pressure.

## Fine-tune

- Added `preview-report` and `download-bundle` actions to the admin fine-tune API.
- Added Markdown report preview with direct bundle download link.
- Added full bundle `.tgz` export for sharing or archiving a finished local training run.
- Added overlay curves for recent runs using the same adapter name.
- Preserved normalized loss display while keeping raw loss values available in tooltips and reports.

## Dataset Presets

Added starter samples for:

- OpenHermes 2.5 style chat data.
- OpenAssistant / OASST-style conversation data.
- Code Alpaca style instruction data.

The validator now accepts and normalizes common community schemas, including `messages`, `conversations`, `turns`, and `instruction` / `output` pairs.

## Compare

- The runtime rail now collapses by default on smaller screens and remains visible on large layouts.
- This keeps the central Compare workspace usable when the viewport is narrow.

- `.env.local` remains ignored and must not be published.
- Full fine-tune bundles can be large because they include adapter artifacts and checkpoint data.
