# v1.1.0-rc.2 Desktop Distribution Gate

Date: 2026-07-16

## English

### What changed

- Replaced the shell-script `CFBundleExecutable` with a compiled arm64 Mach-O launcher. The launcher starts the bundled Node runtime without Terminal, writes its PID and logs to the selected profile, checks `/agent`, and opens the browser only when requested.
- Upgraded the Apple distribution pipeline to sign nested Mach-O code before the app bundle, apply hardened runtime and a minimal Node JIT entitlement, retain secure timestamps, notarize both the app ZIP and final DMG, retain Apple logs, staple both artifacts, and verify them with `codesign`, `stapler`, and Gatekeeper.
- Split Apple `preflightReady` from `completed`. Configured credentials no longer make the GA gate pass before an `Accepted` Apple receipt exists.
- Added a portable clean-machine acceptance runner. It verifies the notarized DMG digest, quarantine/Gatekeeper behavior, read-only mount, Developer ID signature, stapled ticket, isolated-profile boot, Agent/API health, process cleanup, uninstall-preserve, and explicit purge.
- Added organization receipt import with RSA-SHA256 verification and an out-of-band pinned public-key digest. The receipt must bind the exact request, package digest, a different host fingerprint, a durable organization/operator identity, and every required check.

### Current evidence

- `npm run typecheck:changed`: passed all 11 changed partitions before packaging.
- `npm run build:desktop-rc`: built the native-launcher app, ZIP, and DMG.
- `npm run verify:desktop-rc`: passed read-only DMG mount, ad-hoc integrity verification, terminal-free launch, `/agent`, onboarding API, isolated profile, process stop, and detach.
- `npm run rehearse:desktop-external-acceptance`: trusted fixture import passed and a tampered receipt was rejected in an isolated data directory. This is contract evidence only.
- `/api/desktop/onboarding-release`: `8 pass`, `2 watch`, `0 blocked`; local RC is ready and GA remains false.

### External GA procedure

1. Install a valid `Developer ID Application` identity in the release Keychain and store Apple notary credentials with `xcrun notarytool store-credentials`.
2. Start the application with `FIRST_LLM_APPLE_SIGNING_IDENTITY`, `FIRST_LLM_APPLE_NOTARY_PROFILE`, `FIRST_LLM_DESKTOP_APP_PATH`, and `FIRST_LLM_DESKTOP_DMG_PATH`, then POST `/api/desktop/apple-release-signing`.
3. Transfer the resulting notarized DMG, `desktop-external-acceptance-request.json`, and `accept-desktop-release.sh` to a different organization-controlled Mac.
4. Run the acceptance script with an organization-controlled RSA private key. Return only the receipt, signature, and public key.
5. Confirm the public-key SHA-256 through an independent channel and run `npm run import:desktop-acceptance -- ... --expected-key-sha256 <digest>` on the release host.

### Honest gate status

This workstation currently has no valid Developer ID identity, no configured notary profile, and no independent Mac/organization receipt. The production workflow is executable and fail-closed, but `v1.1.0` GA remains on `HOLD` until those external receipts are real.

## 中文

### 本次更新

- 将 shell 脚本形式的 `CFBundleExecutable` 替换为编译后的 arm64 Mach-O launcher。launcher 无需 Terminal 即可启动内置 Node，写入 PID/日志，检查 `/agent`，并仅在需要时打开浏览器。
- Apple 分发链升级为：先签内部 Mach-O，再签 app bundle；启用 hardened runtime 和最小 Node JIT entitlement；保留 secure timestamp；分别公证 app ZIP 与最终 DMG；保存 Apple 日志；对两层产物执行 staple，并通过 `codesign`、`stapler` 和 Gatekeeper 验证。
- 将 Apple `preflightReady` 与 `completed` 分开。仅配置凭据不再导致 GA 门禁提前通过，必须存在 Apple `Accepted` receipt。
- 新增可移交到独立 Mac 的 clean-machine 验收脚本，覆盖 DMG 摘要、quarantine/Gatekeeper、只读挂载、Developer ID、stapled ticket、隔离 profile 启动、Agent/API 健康、进程清理、卸载保留数据和显式清除数据。
- 新增组织签收导入：使用 RSA-SHA256 校验，并要求通过独立渠道固定公钥摘要。receipt 必须绑定原始 request、包摘要、不同宿主指纹、持久组织/operator 身份及完整检查集。

### 当前证据

- `npm run typecheck:changed`：打包前 11 个 changed partitions 全部通过。
- `npm run build:desktop-rc`：已构建原生 launcher app、ZIP 和 DMG。
- `npm run verify:desktop-rc`：只读挂载、ad-hoc 完整性、无 Terminal 启动、`/agent`、onboarding API、隔离 profile、进程停止和镜像卸载全部通过。
- `npm run rehearse:desktop-external-acceptance`：隔离目录中受信 fixture 导入成功，篡改 receipt 被拒绝。该结果只证明契约，不属于外部组织证据。
- `/api/desktop/onboarding-release`：`8 pass`、`2 watch`、`0 blocked`；本地 RC ready，GA 仍为 false。

### 真实外部门禁

当前工作站没有有效 Developer ID 身份、notary profile，也没有独立 Mac/组织签收。生产工作流已经可执行并保持 fail-closed，但在真实外部 receipt 到位前，`v1.1.0` GA 继续保持 `HOLD`。
